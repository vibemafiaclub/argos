## 발견 이슈 (사용자 반영 선택용)

- **[#1] major | `packages/web/src/lib/server/daily-rollup.ts:10` | `SKILL_COUNTS_INVALIDATION_AT = 2026-05-16T00:00:00Z` 가 현재(2026-05-15) 보다 미래라서 갓 upsert 된 row 가 곧바로 stale 판정 → 가드 통과 시점까지 모든 overview/skills 요청이 perpetual lazy recompute (cache 무력화, 1차 요청 ~520ms, 2차도 동일). | 권고: 머지 직후 + 충분한 과거 시각으로 박거나 `Date.now() - 60s` 같은 동적 baseline 사용. 05-review.md 의 ADR-015 가 이미 (Must) 로 등록.**
- **[#2] major | `packages/web/src/lib/server/__fixtures__/skill-call-fixture.ts:79` (and `daily-rollup.test.ts:169`) | `seedSkillCallFixture` 는 events/messages 만 시드하고 project/session/user 행은 안 만든다. JSDoc 은 "이미 DB 에 존재해야 한다" 로 명시했지만 `daily-rollup.test.ts` 의 beforeAll 이 그 supporting record 들을 생성하지 않음 → `events.sessionId_fkey` FK 위반으로 DB 연동 케이스 1건 fail. | 권고: WU-4 fixture 에 `ensureSkillCallFixturePrereqs` 같은 prereq seed (project upsert, session upsert, user upsert) 를 export 추가하거나, beforeAll 에서 `db.project.upsert / db.user.upsert / db.claudeSession.upsert` 명시 호출.**
- **[#3] nit | `packages/web/scripts/invalidate-skill-counts.ts` — 기본 dry-run** | 좋은 안전장치. 다만 runbook(`Decision-9`) 은 그대로 실행하면 안 되고 반드시 `--execute` 가 필요함을 README/PR 본문에 한 줄 명시 권장. | 권고: PR description 의 "배포 step 2/3" 명령에 `--execute` 추가.
