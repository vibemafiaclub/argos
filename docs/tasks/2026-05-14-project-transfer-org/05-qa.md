## 발견 이슈

- [#1] **major** | `packages/web/src/lib/server/project-actions.test.ts:88, 362` | `pnpm typecheck` 실패 — `setupTransactionCallbackRunner` 의 `vi.mocked(db.$transaction).mockImplementation((arg: ((tx: TxClient) => Promise<unknown>) | unknown[]) => ...)` 시그니처가 Prisma 의 `NormalizedPrecedure<{ <P extends Prisma.PrismaPromise<any>[]>(...): ...; <R>(fn: (prisma: Omit<PrismaClient, ...>) => ...): ... }>` 와 호환되지 않음. `TxClient` 가 `$queryRaw` 등 11개 멤버를 빠뜨림. 빌드/테스트는 통과하지만 CI 의 `typecheck` job 이 빨간불. | 권고: `mockImplementation((arg: any) => ...)` 로 캐스팅하거나, 콜백 시그니처를 `Parameters<typeof db.$transaction>[0]` 로 추론. 또는 `setupTransactionCallbackRunner` 내부 `arg` 를 `as unknown as` 거쳐 분기.

- [#2] **nit** | 일반 | dev 서버 첫 콜드 컴파일이 약 2초 걸리며 `/api/events` 첫 호출이 한 번 컴파일되는 지연이 있음. self-heal 자식이 `AbortSignal.timeout(10000)` 로 안전 마진을 확보하므로 시나리오 통과에는 무영향. CI 에는 영향 없음.

이슈 #1 외 critical / major 없음.
